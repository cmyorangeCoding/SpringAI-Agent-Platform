# SpringAI-Agent-Platform

一个基于 Spring AI 的轻量级 Agent 平台练手项目，主要用于学习大模型应用开发流程，包括对话、向量检索、会话记忆等核心能力。

## 📌 项目简介

本项目是个人练手项目，参考黑马相关课程实现，目标是搭建一个最小可用的 AI Agent 系统，帮助理解：

* 大模型调用流程（基于 Spring AI）
* RAG（检索增强生成）基本实现
* 会话上下文管理
* 前后端分离部署方式

整体设计偏学习导向，不追求生产级架构。

---

## 🏗️ 技术架构

### 后端

* **Spring Boot**
* **Spring AI（1.0.0-M6）**
* 支持基础的对话能力
* 集成向量检索（本地向量库）
* 内存级会话记忆（非持久化）

### 前端

* 已构建的静态页面（HTML + JS）
* 使用 **Nginx** 部署
* 与后端通过 HTTP 接口通信

---

## ⚙️ 核心功能

### 1. 基础对话

* 支持与大模型进行多轮对话
* 基于 Spring AI 封装调用

### 2. 会话记忆（Memory）

* 使用内存存储上下文
* 每个会话在服务重启后会丢失
* 适合学习和调试，不适合生产环境

### 3. 向量检索（RAG）

* 向量数据存储在本地
* 支持简单的知识增强问答
* 检索 + 大模型生成的基本流程

### 4. 前后端分离

* 前端为打包后的静态资源
* 后端提供 API
* 通过 Nginx 统一部署

## 🚀 启动方式

### 1. 后端启动

```bash
# 进入后端目录
cd backend

# 启动项目
mvn spring-boot:run
```

或使用 IDE 直接运行主启动类。

---

### 2. 前端部署

将 `frontend` 中的静态文件部署到 Nginx：

```bash
# 示例路径
/usr/share/nginx/html
```

配置好反向代理，将 API 请求转发到后端服务。

## 🧠 设计说明

### 为什么使用内存会话？

* 实现简单
* 方便理解上下文传递
* 适合练手

但缺点是：

* 不可持久化
* 不支持分布式扩展

---

### 为什么向量库存本地？

* 降低依赖（无需引入外部向量数据库）
* 便于快速验证 RAG 流程

后续可以扩展为：

* Redis
* Milvus
* Elasticsearch

* 向量库为本地实现，性能有限
